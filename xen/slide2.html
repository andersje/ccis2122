• multiple Operating Systems, one computer
  • emulation, paravirtualization, full virtualization
  • emulation: interpret hardware calls via a program. No OS modification required. Slow (sometimes)
  • paravirtualization: modify the Operating system.
  • full virtualization: hardware supports virtualization, but OSes must be for same hardware
  • emulation: MAME, bochs, MS Virtual PC, vmware
  • paravirtualization: xen, UML
  • full: xen, VM
  • all can be rebooted/hacked/installed independently
